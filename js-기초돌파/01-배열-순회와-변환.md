# 01. 배열 순회와 변환 — for / forEach / map / filter

> **목표:** "배열을 돌면서 무언가 한다"를 명령형(for)에서 선언형(map/filter)으로 바꿀 수 있다.

## 왜 필요한가
대시보드 카드들은 전부 **배열 → 화면 요소** 변환이다. 일정 리스트, 차트 데이터, 그리드 행 전부.
`schedule.map(s => <li>...)` 가 안 보이면 React 렌더가 막힌다.

---

## 1) 가장 익숙한 것부터 — `for`
```js
const nums = [1, 2, 3];
for (let i = 0; i < nums.length; i++) {
  console.log(nums[i]);
}
```
인덱스(`i`)를 직접 관리. 전통 웹에서 그리드 행 돌릴 때 쓰던 그 방식.
**문제:** 인덱스 실수(off-by-one), 의도가 안 드러남. "1씩 증가시켜 끝까지"라는 잡일을 매번 손으로 씀.

## 2) `forEach` — "그냥 하나씩 실행"
```js
nums.forEach((n) => console.log(n)); // n: 각 원소
nums.forEach((n, i) => console.log(i, n)); // i: 인덱스(필요할 때만)
```
- **반환값 없음(undefined).** 결과를 모으지 않고 **부수효과**(로그 찍기, DOM 조작 등)만 할 때.
- `(n) => ...` 이 **콜백**. "각 원소에 대해 이 함수를 실행해줘"라고 forEach에게 **함수를 값으로 넘기는** 것. ([[콜백]])

## 3) `map` — "변환해서 **새 배열**로"
```js
const doubled = nums.map((n) => n * 2); // [2, 4, 6]
```
- 핵심: **원본은 그대로**, 같은 길이의 **새 배열**을 반환. (1:1 변환)
- `return`한 값이 새 배열에 차곡차곡 담긴다.
- 화살표 함수 한 줄이면 `return` 생략 가능. 중괄호 쓰면 `return` 필요:
```js
nums.map((n) => { return n * 2; }); // 중괄호 → return 명시
nums.map((n) => n * 2);             // 한 줄 → 자동 return
```

**프로젝트 연결:**
```jsx
{schedule.map((s) => <li key={s.id}>{s.title}</li>)}
```
"일정 배열 → li 배열"로 변환. `key`는 React가 항목을 구분하는 용도(원소마다 고유값).

## 4) `filter` — "조건 맞는 것만 **골라낸** 새 배열"
```js
const evens = nums.filter((n) => n % 2 === 0); // [2]
```
- 콜백이 **true면 통과, false면 버림.** 길이가 줄 수 있다.
- map은 "모양 바꾸기", filter는 "걸러내기". 역할이 다름.

## ⭐ 공통 — 콜백은 사실 **3인자** `(원소, 인덱스, 배열)`
`forEach`·`map`·`filter`·`find`·`some`·`every` **전부** 콜백에 같은 3개를 넘긴다.
```js
arr.map((element, index, array) => ...)
//        현재원소   인덱스    원본배열 전체
```
- **필요한 만큼만 받으면 된다.** JS는 "넘기는 인자 ≤ 받는 매개변수"일 필요가 없어서, 안 쓸 뒤 인자는 그냥 안 적는다. (전통 웹처럼 시그니처를 다 맞출 필요 없음)
```js
arr.filter((n) => n > 0)           // 원소만
arr.filter((n, i) => i % 2 === 0)  // 인덱스 필요할 때
arr.filter((n, i, arr) => ...)     // 원본까지 (드뭄)
```
- **세 번째 `array`** 활용 예 — 중복 제거: `arr.filter((v, i, arr) => arr.indexOf(v) === i)` (처음 등장한 것만).

⚠️ **함정:** 콜백에 함수를 **직접** 넘기면 index가 딸려 들어가 사고난다.
```js
["1", "2", "3"].map(parseInt)        // [1, NaN, NaN] ❌  parseInt("2", 1)로 호출돼 진법 꼬임
["1", "2", "3"].map((s) => parseInt(s)) // [1, 2, 3] ✅
```
⚠️ **예외 `reduce`:** 얘만 앞에 acc가 더 붙어 **4인자** → `(acc, 원소, 인덱스, 배열)`. (그래서 02장이 `(acc, n)`)

## 5) 체이닝 — 줄줄이 연결
`map`/`filter`는 **새 배열을 반환**하므로 점(.)으로 이어붙일 수 있다.
```js
const result = nums
  .filter((n) => n % 2 === 1) // 홀수만 [1, 3]
  .map((n) => n * 10);        // [10, 30]
```
"홀수만 → 10배" 가 **읽는 순서대로** 드러남. 이게 선언형의 힘.

## ⚡ 직접 해보기 (답 보기 전에 먼저!)
```js
const members = [
  { name: "채원", age: 34, role: "admin" },
  { name: "민지", age: 31, role: "user" },
  { name: "지호", age: 7,  role: "user" },
];
```
1. 모든 멤버의 **이름만** 모은 배열 `["채원","민지","지호"]` 만들기
2. **성인(age >= 20)** 만 골라내기
3. 성인들의 이름 뒤에 `"님"`을 붙인 배열 만들기 (`["채원님","민지님"]`) — 체이닝으로
4. (생각만) `forEach`로 3번을 하려면 뭐가 불편할까?

---

## ✅ 체크포인트
- map은 **새 배열**(길이 같음), filter는 **새 배열**(길이 줄 수 있음), forEach는 **반환 없음**.
- 콜백 = "각 원소마다 실행할 함수를 값으로 넘긴다."
- 한 줄 화살표함수는 `return` 자동, 중괄호 쓰면 `return` 직접.

```js
// 1
members.map((m) => m.name);

members.filter((m) => m.age >= 20).map((m) => m.name + "님");

```
