## 01. 배열 순회와 변환 — 2026-06-23

**총평: 4/4 (이해도 만점).** 고칠 점은 1번 변수명 오타 하나.

| # | 제출 | 결과 |
|---|------|------|
| 1 | `member.map((obj) => obj.name)` | 🟡 로직 정답, **오타**: `member` → `members` (배열명). 그대로면 `member is not defined` |
| 2 | `members.filter((obj) => obj.age >= 20)` | ✅ 정답 |
| 3 | `members.filter((obj) => obj.age >= 20).map((adult) => adult.name + "님")` | ✅ 정답. 콜백명을 `adult`로 둬 의미가 드러남 |
| 4 | "담을 배열 미리 선언 + 내부 조건문/push로 복잡, **반환이 없어서**" | ✅ 정답. forEach 한계 핵심 정확 |

**짚을 점**
- 배열 변수명은 **복수형 `members`** — 오타는 실무에서 제일 잦은 실수라 의식적으로.
- 3번 체이닝 이해 OK: `filter`가 새 배열을 반환 → 그 위에 `.map` 연결되는 원리 그대로 적용함.

## 02. 배열 집계 reduce — 2026-06-23

**총평: 4/4 (이해도 만점) 🎉** 어제 막혔던 그룹핑을 혼자 재현. 핵심 통과.

| # | 제출 | 결과 |
|---|------|------|
| 1 | `expense.reduce((arr, obj) => arr + obj.amount, 0)` | ✅ 정답. 누적값명 `arr`→숫자라 `acc`/`sum` 권장(가독성) |
| 2 | `arr[obj.category] ??= 0; arr[obj.category] = obj.amount + arr[obj.category]; return arr` (초기값 `{}`) | ✅ 정답. `??=` 방식이 모범답안보다 깔끔 |
| 3 | `arr[obj.category] ??= []; arr[obj.category].push(obj); return arr` (초기값 `{}`) | ✅ 정답. **이번 장 핵심** 스스로 재현 |
| 4 | "map은 누적 아님 / filter는 길이만 줆 / reduce는 누적값(acc)을 인자로 받아 누적" | ✅ 정답. 세 메서드 차이 정확 |

**짚을 점**
- **`expense` → `expenses`** (배열명 오타). 01장 `member`에 이어 **2회 반복** → 복수형 배열명 의식하기.
- 누적값 이름: 숫자면 `acc`/`sum`, 객체/배열이면 `acc`로 일관되게. (`arr`는 "배열"로 오해 소지)
- 잘한 점: `??= 0` / `??= []` 초기화 패턴을 합계·그룹핑 양쪽에 자연스럽게 적용함.

## 03. 배열 탐색·정렬 — 2026-06-24

**총평: 2/4.** 1·4번 OK. 2번 조건 반대, 3번 원본 mutate(=4번과 모순) — **불변성**이 핵심 미스.

| # | 제출 | 결과 |
|---|------|------|
| 1 | `members.find((val) => val.name === "민지")` | ✅ 정답 |
| 2 | `members.some((val) => val.age > 20)` | ❌ **조건 반대**. 미성년자 = `age < 20` (성인 `> 20` 검사함). 메서드 선택은 맞음 |
| 3 | `const age = members.sort((a,b) => b.age - a.age)` | 🟡 비교함수 맞음. 근데 **`[...members]` 복사 안 함 → 원본 mutate**. 변수명 `age`도 어색(`sorted`) |
| 4 | "불변성 못 지킴 / 객체참조로 값 변경 / 새 배열로 받아야" | ✅ 핵심 정확 (용어만: "구조분해"→**스프레드** `[...]`) |

**짚을 점**
- **3↔4 모순**: 4번에서 "새 배열로 받아야"라 쓰고 3번은 원본 직접 sort. 머리는 알아도 손이 안 따라옴 → `const sorted = [...members].sort(...)`.
- `sort`/`reverse`/`splice`는 **원본 mutate** (map/filter와 다름). state엔 항상 `[...arr].sort()`.
- 용어: **스프레드**(`...`)=펼쳐 담기 / **구조분해**=꺼내 받기. (05장에서 확실히 갈림)

## 04. 대입과 단축평가 — 2026-06-24

**총평: 6·7·8 만점, 5번 예측 3/5.** 틀린 2개가 같은 뿌리 — `||`도 0/""를 통과시킨다는 오해.

| # | 제출 | 결과 |
|---|------|------|
| 5 | `0, 0, "x", "x", "x"` | 🟡 3/5. `0\|\|"기본"`→**"기본"**(0은 falsy) ❌, `""??"x"`→**""**(null/undef 아님) ❌. 나머지 ✅ |
| 6 | `undefined` | ✅ 정답 (`obj.c` 없어서 멈춤) |
| 7 | `??` | ✅ 정답 (score 0 살려야 함) |
| 8 | `if(acc[key]===null\|\|acc[key]===undefined) acc[key]=0; acc[key]=acc[key]+1` | ✅ **모범답안보다 정확** — `??`가 검사하는 null/undefined를 명시적으로 풀어씀 |

**짚을 점**
- 핵심 오해: **`||`는 falsy 7개 전부**에서 오른쪽 / **`??`는 null·undefined 둘만**. `0??x`=0은 맞췄는데 `0||x`도 0이라 착각 → `||`도 0/""를 떨군다는 걸 놓침.
- 외우기: `||`="비어있으면(falsy) 기본값", `??`="**아예 없을 때만**(null/undef) 기본값". 숫자0·빈문자 살리려면 `??`.
- 잘한 점: `?.`(6), `??` 선택(7), `??=` 풀어쓰기(8) — 단축평가 **원리**는 완전히 잡음.

## 05. 구조분해·스프레드·rest — 2026-06-29

**총평: 결론 전부 정답, 3번만 문법 오류(`=`→`:`).** 로직 이해는 완성, 객체 리터럴 기호 + 철자가 고칠 점.

| # | 제출 | 결과 |
|---|------|------|
| 1 | `const { name, age } = memeber;` | 🟡 로직 정답. **오타** `memeber`→`member` |
| 2 | `const [ first, ...others ] = list;` | ✅ 정답. rest로 나머지 모으기 정확 |
| 3 | `const updateMemeber = { ...memeber, age = 32 };` | ❌ **문법 오류**: 객체 리터럴은 `=` 아니라 **`:`** → `{ ...member, age: 32 }`. 스프레드+덮어쓰기 의도는 맞음 |
| 4 | `const udpateList = [ ...list, "e" ];` | ✅ 정답. 변수명 `udpate` 오타만 사소 |
| 5 | "원본으로 새 객체 만들고, React 객체 불변성 지켜야 해서" | ✅ 정답. 핵심 정확 |

**짚을 점**
- **3번 핵심**: 객체 `{}` 안 키-값은 **`key: value`(콜론)**. `=`는 대입/기본값 매개변수용 → 객체 안에 쓰면 SyntaxError. (`age = 32` ❌ / `age: 32` ✅)
- **철자 반복**: `memeber`(member). 01 `member→members`, 02 `expense→expenses`에 이어 **3회째** → 변수명 철자 의식적으로 점검(실무 `is not defined` 1순위 실수).
- 5번 보강: 원본 mutate하면 React가 `===`(주소 비교)로 변경 못 감지 → 새 객체(새 주소)여야 리렌더. ([[07-값과-참조]] 원리와 연결)
- 잘한 점: rest(2), 스프레드 덮어쓰기 의도(3), 배열 추가(4) — 구조분해↔스프레드 **용어/방향 혼동 없이** 분리해서 씀(05장 목표 달성).

## 06. 변수와 스코프 — 2026-06-29

**총평: 4/5.** 2·3·4·5 정확(블록스코프·const 재할당 완벽). 약점 = **1번 `var` 호이스팅**(`10`으로 예측, 정답 `undefined`).

| # | 제출 | 결과 |
|---|------|------|
| 1 | `10` | ❌ 정답 **`undefined`**. var는 **선언만** 호이스팅, 값 할당은 제자리 → `console.log` 시점엔 아직 undefined |
| 2 | `ReferenceError` | ✅ 정답 (let 블록스코프 → 블록 밖 안 보임) |
| 3 | `2`, 값 변경됨 | ✅ 정답 (const 객체 내부 변경 가능) |
| 4 | "재할당 불가, 내부 추가/변경/삭제 가능" | ✅ 정답 (`list=[3,4]` ❌ / `list.push(3)` ⭕) |
| 5 | "let이 블록스코프라서" | ✅ 정답 |

**짚을 점**
- **핵심 오해(1번)**: "할당까지 위로 올라간다"고 봄. 실제로 호이스팅되는 건 **선언(`var x`)뿐**, `x = 10`은 제자리 → console.log 시점엔 undefined.
- **1↔2 대비가 교훈**: 같은 "선언 전 사용"인데 var=`undefined`(조용한 버그) / let=`ReferenceError`(명확). → **var를 안 쓰는 이유** 그 자체.
- 잘한 점: const 재할당 vs 내부 변경 구분(4), 블록스코프 판단(2·5) 정확.

## 07. 값과 참조 — 2026-06-29

**총평: 개념 5/5 🎉** (얕은/깊은 복사 사전 Q&A 효과). 흠은 5번 **용어**뿐 — `{...a}`를 "구조분해"라 부름(→ 스프레드).

| # | 제출 | 결과 |
|---|------|------|
| 1 | `10`, 원시값은 그 자체를 담아 안 변함 | ✅ 정답 |
| 2 | `99`, 같은 참조 위치 가리켜 원본 변경 | ✅ 정답 |
| 3 | 모두 False, 참조 위치 다름 | ✅ 정답 |

## 09. 템플릿 리터럴·매개변수 — 2026-06-29

**총평: 5/5.** 템플릿·기본값·rest/스프레드·구조분해 매개변수 전부 정확. React props 연결까지 스스로 짚음.

| # | 제출 | 결과 |
|---|------|------|
| 1 | `` `${member.name}(${member.age})` `` | ✅ "민지(31)" |
| 2 | `9, 8` | ✅ 기본값 매개변수(`exp=2`) 정확 |
| 3 | `function max(...nums){ return Math.max(...nums) }` | ✅ **rest(모으기) + 스프레드(펼치기)** 한 줄에 둘 다, 방향 구분 정확 |
| 4 | `function intro({name, age}){ return `${name}는 ${age}살` }` | ✅ 구조분해 매개변수 |
| 5 | "React가 객체로 묶어 넘기고 파라미터단에서 구조분해해서 편함" | ✅ 정답. 핵심(props=구조분해 매개변수) 잡음 |

**짚을 점**
- 3번 우수: 같은 `...`인데 **매개변수 자리=rest(모으기) / 호출 인자 자리=스프레드(펼치기)** 를 자연스럽게 둘 다 사용.
- 5번 보강(2가지 구체 이득): ① 함수 **입구만 봐도 뭘 받는지** 보임 ② 본문에서 `props.`/`member.` 안 붙임. → React 함수 컴포넌트가 `({...})` 형태인 이유.
- **9강 `intro({name,age})` = React 컴포넌트 시그니처 그 자체** → react-실전 트랙 직전 워밍업 완료.
