# 07. 값과 참조 — 원시 vs 객체, `=`가 복사가 아닌 이유, `===`, 얕은/깊은 복사

> **목표:** 03강에서 막혔던 **불변성**의 뿌리 — "왜 `[...arr]`로 복사해야 하나"를 값/참조로 이해한다.

## 왜 필요한가
03강 채점에서 `sort`를 `[...members]` 없이 원본에 직접 걸어 mutate한 게 약점이었다.
그 "왜 복사해야 하나"의 답이 **값 vs 참조**다. React state 불변성도 전부 여기서 나온다.

---

## 1) 두 부류 — 원시값 vs 객체
- **원시값(primitive):** `number` `string` `boolean` `null` `undefined` `symbol` `bigint` → **값 그 자체**를 담는다.
- **객체(object):** 객체·배열·함수 → 변수엔 **값이 아니라 "값이 있는 곳 주소(참조)"** 가 담긴다.

```js
let a = 1;
let b = a;     // 값 1을 복사
b = 2;
console.log(a); // 1  ← a는 영향 없음 (각자 값)

let x = { n: 1 };
let y = x;      // 주소를 복사 → 같은 객체를 가리킴
y.n = 2;
console.log(x.n); // 2  ← x도 바뀜! (같은 걸 보고 있음)
```

**전통 웹 비유:** 원시값 = 종이에 숫자를 **베껴 적기**(각자 사본). 객체 = **같은 문서의 링크를 공유**(한 명이 고치면 다 보임).

## 2) `=`는 "복사"가 아니다 (객체일 때)
`y = x`는 **내용을 복제**하는 게 아니라 **같은 주소를 가리키게** 할 뿐. 그래서 한쪽을 mutate하면 둘 다 바뀐 것처럼 보인다.
→ 03강에서 `sort`가 원본까지 바꾼 이유: `sort`는 **원본 배열 자체**를 정렬한다(새 배열 안 만듦).

## 3) `===` 비교 — 원시는 값, 객체는 주소
```js
1 === 1            // true  (값 비교)
"a" === "a"        // true

{} === {}          // false ← 내용 같아도 다른 주소(서로 다른 객체)
[1] === [1]        // false
const o = {}; o === o // true  (같은 주소)
```
- 객체는 **내용이 아니라 "같은 객체냐(주소)"** 로 비교한다.
- ⭐ **React가 변경을 감지하는 방식이 바로 이 `===`(주소 비교)다.** state를 **새 객체/배열로 교체**해야 "주소가 달라졌네 = 바뀌었네" 하고 리렌더한다. 원본을 mutate하면 주소가 그대로라 **못 알아챈다.**

## 4) 얕은 복사 — 한 겹만 새로
원본을 안 건드리고 새 걸 만들려면 **스프레드**([[05-구조분해-스프레드-rest]]):
```js
const arr = [3, 1, 2];
const sorted = [...arr].sort((a,b)=>a-b);  // 원본 보호하고 정렬 (03강 정답)
console.log(arr);    // [3,1,2] 그대로

const user = { name: "채원", age: 34 };
const updated = { ...user, age: 35 };       // 새 객체, age만 교체
```
- `[...arr]` / `{...obj}` = **바깥 한 겹**을 새 주소로 복사.

## 5) ⚠️ 얕은 복사의 한계 — 중첩은 공유된다
```js
const a = { name: "채원", addr: { city: "서울" } };
const b = { ...a };          // 한 겹만 복사
b.addr.city = "부산";
console.log(a.addr.city);    // "부산" ← 안쪽 addr는 같은 주소를 공유!
```
- 스프레드는 **한 겹**만 새로. 안쪽 객체는 여전히 같은 참조.
- 깊은 곳까지 복사하려면 **`structuredClone(a)`**(깊은 복사) — 자세힌 [[개념파악]]/25장. state도 중첩이 깊으면 안쪽까지 새로 만들어야 한다.

## ✅ 체크포인트
- 원시값 = 값 복사(독립), 객체/배열 = **주소(참조) 복사**(공유).
- `=`는 객체를 복제하지 않는다. mutate하면 같이 바뀜 → 03강 `sort` 사고의 원인.
- `===`는 객체를 **주소로** 비교. **React 변경 감지가 이 원리** → state는 새 값으로 교체.
- 스프레드 = **얕은(한 겹)** 복사. 중첩은 `structuredClone`.
