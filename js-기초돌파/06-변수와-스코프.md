# 06. 변수와 스코프 — `var` `let` `const`, 블록 스코프, 호이스팅, TDZ

> **목표:** 왜 `const`를 기본으로 쓰고 `var`는 안 쓰는지, "선언했는데 왜 에러?"가 왜 나는지 원리로 안다.

## 왜 필요한가
네 코드는 거의 `const`다(`const group = ...`, `const [schedule, setSchedule] = ...`). 가끔 `let rowData`도 보임.
**언제 `const`/`let`을 쓰고, `var`는 왜 안 쓰나**를 알아야 React 버그(특히 반복문 안 클로저)의 숨은 원인을 피한다.

---

## 1) 세 가지 차이 한 장 요약

| | 재할당 | 스코프 | 호이스팅 | 비고 |
|---|---|---|---|---|
| `const` | ❌ 불가 | **블록** `{}` | TDZ | **기본값** |
| `let` | ⭕ 가능 | **블록** `{}` | TDZ | 바꿔야 할 때만 |
| `var` | ⭕ 가능 | **함수** | 끌어올려져 `undefined` | **쓰지 마라(레거시)** |

```js
const PI = 3.14;
PI = 3;         // ❌ TypeError: Assignment to constant variable

let count = 0;
count = 1;      // ⭕
```

## 2) 블록 스코프 — `{}` 안에서만 산다
```js
if (true) {
  const x = 1;
  var y = 2;
}
console.log(x); // ❌ ReferenceError (블록 밖이라 안 보임)
console.log(y); // 2  ← var는 블록을 안 가리고 새어나감(함수 스코프)
```
- `let`/`const`는 **`{}` 단위**로 갇힌다(if·for·함수 다 블록).
- `var`는 블록을 무시하고 **함수 전체**로 퍼진다 → 의도 안 한 곳에서 보임 = 버그.

**전통 웹 비유:** 옛날 JS는 `var`뿐이라, `for` 루프의 `var i`가 루프 밖까지 살아남아 사고가 잦았다. `let`이 그걸 블록에 가둬 고쳤다.

## 3) 호이스팅 — 선언이 위로 "끌어올려진" 것처럼 보임
```js
console.log(a); // undefined  (var는 선언만 위로, 값은 아직)
var a = 1;

console.log(b); // ❌ ReferenceError  (let/const는 TDZ)
let b = 1;
```
- 자바스크립트는 코드 실행 전에 **선언을 먼저 훑는다**(호이스팅).
- `var`: 선언이 끌어올려지고 **`undefined`로 초기화** → 위에서 써도 에러는 안 나고 `undefined`(더 헷갈림).
- `let`/`const`: 끌어올려지긴 하나 **선언 줄 전까지 "쓰면 안 됨" 구역** = **TDZ(Temporal Dead Zone)**. 그래서 위에서 쓰면 **ReferenceError**(차라리 명확).

## 4) `const`는 "불변"이 아니다 — "재할당 불가"다 ⚠️
```js
const user = { name: "채원" };
user.name = "민지";   // ⭕ OK! 객체 "내용"은 바뀜
user = {};            // ❌ 재할당만 막힘

const arr = [1, 2];
arr.push(3);          // ⭕ [1,2,3]  내용 변경 가능
```
- `const`가 잠그는 건 **변수 ↔ 값의 연결**(다른 걸 가리키게 못 함)이지, **값의 내용**이 아니다.
- 그래서 객체/배열은 `const`여도 속을 바꿀 수 있다. **진짜 불변**(안 바꾸기)은 다음 [[07-값과-참조]]·스프레드의 영역.

## 5) 실무 규칙
- **기본 `const`.** 재할당이 꼭 필요할 때만 `let`. `var`는 **안 쓴다**(레거시 코드 읽을 때만 의미).
- 네 코드 `let rowData = [...]`도 재할당 안 하면 `const`가 맞다(린트가 종종 잡아줌).

## ✅ 체크포인트
- 기본 `const`, 바꿀 때만 `let`, `var`는 안 씀(함수 스코프 + undefined 호이스팅이 함정).
- `let`/`const`는 **블록 스코프** + **TDZ**(선언 전 사용 시 ReferenceError).
- `const` = **재할당 금지**일 뿐, 객체/배열 **내용은 바뀐다**.
