# 04. 대입과 단축평가 — `??` `||` `&&` `??=` `||=` `&&=` `?.`

> **목표:** 어제 본 `??`, `??=`, `link && ...` 가 왜 그렇게 동작하는지 **원리**로 이해한다.

## 왜 필요한가
어제 코드에 이런 게 잔뜩 나왔다:
- `acc[s.date] ??= []` (그룹핑)
- `{0:"Today", 1:"내일"}[diff] ?? "월 일"` (lookup + 기본값)
- `link && <a href={link}>` (조건부 렌더)
- `params.data?.name` (안전한 접근)

전부 **단축평가(short-circuit)** 라는 한 원리에서 나온다.

---

## 0) 먼저 — truthy / falsy
JS는 모든 값을 참/거짓으로 취급할 수 있다.
- **falsy 7가지:** `false`, `0`, `""`(빈문자열), `null`, `undefined`, `NaN`, `0n`
- 나머지는 전부 **truthy** (`"0"`, `[]`, `{}` 도 truthy!)

## 1) `||` (OR) — "왼쪽이 falsy면 오른쪽"
```js
const name = userName || "이름없음"; // userName이 ""/null/undefined면 "이름없음"
```
- 왼쪽이 **truthy면 왼쪽**을 반환(오른쪽은 평가도 안 함=단축), falsy면 오른쪽.
- **함정:** `0`이나 `""`도 falsy라 의도치 않게 기본값으로 넘어감. (가격 0원인데 기본값 떠버림)

## 2) `??` (널 병합) — "왼쪽이 null/undefined일 때만 오른쪽"
```js
const count = value ?? 0; // value가 null이나 undefined일 때만 0
```
- `||`와 거의 같지만 **falsy 중 `null`/`undefined`만** 따진다. `0`/`""`는 통과시킴.
- **그래서 숫자/문자 기본값엔 `??`가 안전.** 어제 `acc[f] ?? 0` 이 이래서 맞음.

| 입력 | `x \|\| "기본"` | `x ?? "기본"` |
|------|---------|---------|
| `0` | "기본" ❌ | `0` ✅ |
| `""` | "기본" ❌ | `""` ✅ |
| `null` | "기본" | "기본" |

## 3) `&&` (AND) — "왼쪽이 truthy면 오른쪽"
```js
const x = isLoggedIn && userName; // 로그인했으면 userName, 아니면 false
```
- 왼쪽이 **falsy면 왼쪽**에서 멈춤, truthy면 오른쪽을 반환.
- **React 조건부 렌더의 정체:**
```jsx
{link && <a href={link}>화상</a>}
```
"`link`가 있으면(truthy) `<a>`를 렌더, 없으면 `link`(falsy)" → React는 false/null/undefined를 **아무것도 안 그림.**
- ⚠️ 함정: `{count && <p>...}` 에서 count가 `0`이면 화면에 **0이 찍힘**(0은 안 그려지지만 숫자 0 자체가...). 개수 조건은 `count > 0 && ...` 로.

## 4) 단축 대입 — `??=` `||=` `&&=`
"검사 후 대입"을 한 글자로:
```js
acc[date] ??= [];   // acc[date]가 null/undefined면 [] 대입  (어제 그룹핑)
config ||= {};      // config가 falsy면 {} 대입
user &&= user.name; // user가 truthy면 user.name 대입
```
풀어쓰기:
```js
acc[date] ??= [];          //  === if (acc[date] == null) acc[date] = [];
config   ||= {};           //  === config = config || {};
```

## 5) `?.` (옵셔널 체이닝) — "있으면 타고, 없으면 undefined"
```js
params.data?.name;      // data가 null/undefined면 에러 대신 undefined
user?.address?.city;    // 중간이 없어도 안전
arr?.[0];               // 배열도 가능
fn?.();                 // 함수 있으면 호출
```
- 없는 객체의 속성에 접근하면 보통 **TypeError로 터진다.** `?.`는 중간이 null/undefined면 **거기서 멈추고 undefined** 반환.
- 자주 짝: `data?.name ?? "기본값"` (안전 접근 + 기본값).

## 6) lookup 객체 + `??` (어제 dateLabel 패턴)
```js
const label = { 0: "Today", 1: "내일" }[diff] ?? `${month}월 ${day}일`;
```
- `{...}[diff]` = 객체에서 `diff` 키를 꺼냄. 0이면 "Today", 1이면 "내일", **그 외엔 undefined**.
- `?? "..."` 로 undefined일 때 기본 문자열. → if/switch 없이 **매핑 + 기본값**.

## ✅ 체크포인트
- `||` = falsy면 오른쪽 / `??` = **null·undefined만** 오른쪽 (0·"" 살림).
- `&&` = falsy면 멈춤 → **조건부 렌더의 원리.**
- `??=`/`||=`/`&&=` = "검사 후 대입" 단축.
- `?.` = 중간이 없어도 에러 없이 undefined. 짝꿍은 `?? 기본값`.
