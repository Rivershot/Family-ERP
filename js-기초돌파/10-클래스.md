# 10. 클래스 — `class`, 상속(`extends`/`super`), `#private`, `static`, Error 상속

> **목표:** 라이브러리·레거시 코드의 `class`를 막힘 없이 **읽고**, 커스텀 에러를 만든다.
> *요즘 React는 함수+훅이라 직접 작성은 드물다. 이 장은 **읽기 + 에러 클래스**가 핵심.*

## 왜 필요한가
- AG-Grid·ECharts·React 내부, 레거시 **클래스 컴포넌트**가 `class X extends Y` 로 쓰여 있다 → 읽으려면 필요.
- 비동기 에러 처리([[react-실전]] 21장)에서 **커스텀 에러 클래스**(`class ApiError extends Error`)를 직접 만든다.

---

## 1) 기본 — `class` + `constructor` + 메서드
```js
class Member {
  constructor(name, age) {   // new 할 때 자동 실행, 초기 세팅
    this.name = name;        // 인스턴스 속성 (this = 만들어지는 객체)
    this.age = age;
  }
  greet() {                  // 메서드 (프로토타입에 한 번만 저장 — 인스턴스마다 복제 X)
    return `${this.name}(${this.age})`;
  }
}

const m = new Member("채원", 34);  // new로 인스턴스 생성
m.greet();   // "채원(34)"
m.name;      // "채원"
```
- `new` → 빈 객체 만들고 → `constructor` 실행하며 `this`에 속성 채움 → 그 객체 반환.
- 메서드 안 `this` = **그 인스턴스**. ([[08-함수-깊이파기]]의 this — 점 앞 객체가 호출자)

**전통 OOP 비유:** Java의 클래스/생성자/`new`와 거의 같은 그림. 단 JS 클래스는 사실 **프로토타입 위의 문법 설탕**(겉모습만 클래스).

## 2) 클래스 필드 & getter/setter (⚪ 가볍게)
```js
class Counter {
  count = 0;                 // 클래스 필드(ES2022) — constructor 없이 초기값
  get double() { return this.count * 2; }   // 호출 아닌 속성처럼: c.double
  set value(v) { this.count = v; }          // c.value = 5
}
```

## 3) `#private` — 진짜 외부 차단 (ES2022)
```js
class Account {
  #balance = 0;              // # 붙이면 클래스 밖에서 접근 불가
  deposit(n) { this.#balance += n; }
  get balance() { return this.#balance; }
}
const a = new Account();
a.deposit(100);
a.balance;     // 100
a.#balance;    // ❌ SyntaxError (밖에서 못 봄)
```
- [[08-함수-깊이파기]] 클로저로도 은닉했지만, 클래스에선 **`#`** 가 공식 방법.

## 4) 상속 — `extends` / `super`
```js
class Animal {
  constructor(name) { this.name = name; }
  speak() { return `${this.name} 소리`; }
}
class Dog extends Animal {       // Animal을 물려받음
  constructor(name) {
    super(name);                 // ⭐ 부모 constructor 먼저 호출(필수)
    this.legs = 4;
  }
  speak() { return `${this.name} 멍멍`; }  // 같은 이름이면 덮어씀(오버라이드)
}
new Dog("바둑이").speak();   // "바둑이 멍멍"
```
- `extends` = 부모 기능을 물려받기. `super(...)` = 부모 constructor 호출(자식 constructor에서 `this` 쓰기 전에 반드시).
- `super.speak()` 로 부모 메서드도 부를 수 있음.

## 5) `static` — 인스턴스 말고 클래스 자체에
```js
class MathUtil {
  static PI = 3.14;
  static square(n) { return n * n; }
}
MathUtil.square(5);  // 25  (new 없이 클래스에서 바로)
```
- 인스턴스 공통 도구·상수에. (`Array.from`, `Object.keys` 같은 게 static 메서드)

## 6) ⭐ 실전 — 커스텀 에러 클래스 (Error 상속)
```js
class ApiError extends Error {
  constructor(message, status) {
    super(message);            // Error의 message 세팅
    this.name = "ApiError";
    this.status = status;      // 추가 정보(HTTP 상태 등)
  }
}

try {
  throw new ApiError("로그인 실패", 401);
} catch (e) {
  if (e instanceof ApiError) console.log(e.status, e.message); // 401 "로그인 실패"
}
```
- axios 실패를 의미 있는 에러로 감싸 던질 때 쓴다(21장에서 `try/catch`와 함께).
- `instanceof` = "이 클래스(또는 부모)의 인스턴스냐" 판별.

## ⚡ 직접 해보기
1. `class Product` — `constructor(name, price)`로 속성 받고, `info()`가 `"우유: 2000원"` 반환하게. `new Product("우유", 2000).info()`
2. `class Expense extends Product` — `category`를 추가로 받기(`super` 사용). `new Expense("우유", 2000, "식비")`
3. `class Wallet` — `#money = 0` 비공개 + `add(n)` + `get balance()`. 밖에서 `#money` 직접 접근하면?
4. `class ValidationError extends Error` — `field`를 추가로 갖게 만들고 `throw` 후 `catch`에서 `e.field` 출력
5. (생각) 요즘 React가 클래스 컴포넌트 대신 함수+훅을 쓰는데, 그래도 클래스를 알아야 하는 이유 2가지?

---

## ✅ 체크포인트
- `class` + `constructor` + `new`. 메서드 안 `this` = 그 인스턴스.
- `extends`/`super`(부모 호출), 같은 이름 메서드 = 오버라이드.
- `#필드` = 진짜 비공개(ES2022), `static` = 클래스 자체에.
- 실전 = **`class X extends Error`** 커스텀 에러 + `instanceof` 판별.
