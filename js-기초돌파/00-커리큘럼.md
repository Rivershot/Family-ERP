# 모던 JS 기초돌파 커리큘럼

> 프로젝트(Family ERP)에서 막힘의 8할은 React/TS가 아니라 **모던 JS(ES2015~2021) 기본기**였다.
> 특히 **배열 메서드**와 **대입/단축평가 연산자**. 여기를 뚫으면 대시보드·가계부 로직이 술술 풀린다.

## 이 커리큘럼의 사용법

- **읽기만 하지 말고 손으로 친다.** 각 장의 `⚡ 직접 해보기`는 답을 보기 전에 먼저 풀어본다.
- 실습 환경: 브라우저 콘솔(F12) 또는 [playground](https://playcode.io) 아무거나. `console.log`로 결과 찍어보며 진행.
- **전통 웹(WebSquare 등) 비유**를 군데군데 달아둠 — 익숙한 개념에 새 문법을 붙여서 이해.
- 막히면 그 장 번호와 함께 질문. (예: "03장 reduce 초기값이 왜 필요한지 모르겠어")

# 📚 확장 로드맵 — ES6 그 이후

> **너의 출발점:** ES6(2015)까지. 그 이후 ES2016~**2025** 문법·내장 객체를 "이건 뭐지?" 없이 읽고 쓰는 게 목표.
> **설계 방향:** 위 01~05(=1단계)로 어제 막힌 곳을 직격한 뒤, **토대 → 모던 객체 → 비동기 → 모듈** 순으로 넓혀간다.
> **진행 방식:** 한 번에 다 안 만든다. 네가 한 단계 끝낼 때마다 다음 장을 **같은 형식**(왜→개념→비유→`⚡ 직접 해보기`→접힌 정답)으로 작성. 막히면 장 번호로 질문.
>
> **깊이 원칙:** 각 장은 우선 **"실무에서 읽고 쓸 만큼"** — 코드 읽다 안 막히고 직접 쓰는 수준까지. 그 아래 **원리**(클로저가 메모리에 어떻게 남나, 이벤트 루프 큐 순서, 반복 프로토콜 내부 등)는 ⛏️ 표시한 장에서 **"더 파기" 코너**로 따로 둔다 — 처음엔 건너뛰고, 한 바퀴 돈 뒤 또는 막히는 순간 깊게. (실무 감각 먼저, 원리는 선택·심화로 유도)

## 단계별 구성

### 1단계 — 데이터 다루기 기본기 ✅ (작성 완료, 01~05)
어제 막힌 배열·단축평가·구조분해. **여기부터 한다.**
> 01 순회/변환 · 02 reduce · 03 탐색/정렬 · 04 단축평가 · 05 구조분해/스프레드

### 2단계 — JS 토대 재정비 (ES5~ES6, 의외의 구멍 메우기)
"안다고 생각했지만 사실 헷갈리는" 핵심. React 버그의 숨은 원인 대부분 여기.
- [ ] 06 **변수와 스코프** — `var`/`let`/`const`, 블록 스코프, 호이스팅, TDZ, 재할당 vs 불변
- [ ] 07 **값과 참조** — 원시 vs 객체, 복사(`=`가 복사가 아닌 이유), `===` 비교, 얕은/깊은 복사
- [ ] 08 ⛏️ **함수 깊이 파기** — 선언/표현식/화살표, **`this`**(화살표가 다른 이유)·`call`/`apply`/`bind`, **클로저**, 고차함수 〔더 파기: 클로저가 메모리에 남는 원리, this 4가지 바인딩 규칙〕
- [ ] 09 **템플릿 리터럴 · 매개변수** — `` `${}` ``·태그드 템플릿, 기본값 매개변수, 나머지(rest) 매개변수, 구조분해 매개변수
- [ ] 10 **클래스** — `class`/생성자, 상속(`extends`/`super`), `#private`(ES2022)·`static`·getter/setter, **Error 상속**. *React 클래스 컴포넌트·라이브러리 코드 **읽기**용 (요즘 직접 작성은 적음)*
- [ ] 11 ⛏️ **Symbol · 이터레이터 · 제너레이터** — `for...of`/스프레드가 **왜 되는지**(반복 프로토콜), `Symbol.iterator`, 제너레이터(`function*`) 맛보기 〔더 파기: 이터레이션 프로토콜 내부, lazy 평가〕

### 3단계 — 모던 객체 & 자료구조 (ES2017~2025 핵심)
프로젝트 데이터 가공에 직결. 특히 가계부 포맷·집계.
- [ ] 12 **객체 메서드** — `Object.keys/values/entries/fromEntries/assign/hasOwn`, 동적 키, 단축 속성
- [ ] 13 **Map · Set** — 객체 대신 언제 쓰나, 중복 제거(`new Set`), 키-값 순회, **Set 메서드 `union`/`intersection`/`difference`(ES2025)**, `WeakMap` 맛보기
- [ ] 14 **배열 신메서드** — `flat`/`flatMap`(ES2019), `at`(ES2022), `findLast`(ES2023), **`toSorted`/`toReversed`/`with`**(ES2023 불변 버전 ⭐), `Array.from`/`of`
- [ ] 15 **그룹핑 & 지연 순회** — **`Object.groupBy`/`Map.groupBy`(ES2024)** ⭐ ← 02장 `reduce`+`??=`로 손수 짠 **날짜별 그룹핑을 한 줄로** 재방문, 이터레이터 헬퍼 `.map`/`.filter`/`.take`(ES2025)
- [ ] 16 **문자열·숫자** — `padStart`/`padEnd`, `replaceAll`(ES2021), `at`, `1_000_000`(숫자 구분자), `Number`/`parseInt` 차이
- [ ] 17 **Date · Intl** — Date 함정(0-base 월 등), `Intl.NumberFormat`(통화 콤마), `Intl.DateTimeFormat`(날짜 포맷) ← 가계부/일정 직결
- [ ] 18 **JSON** — `JSON.parse`/`stringify`, `replacer`/`reviver`, 깊은 복사 함정(`JSON.parse(JSON.stringify())`의 한계 → `structuredClone`은 25장). axios·mock 데이터 가공 직결

### 4단계 — 비동기 (axios 데이터 fetch의 토대, 이미 실전 맛봄 → 원리 메우기)
1순위 "데이터를 axios로 가져오기"는 2026-06-23 완료. 이제 **왜 그렇게 동작하나**를 채운다.
- [ ] 19 ⛏️ **동기 vs 비동기** — 이벤트 루프 직관, 콜백 지옥이 왜 생겼나 〔더 파기: 콜스택·태스크/마이크로태스크 큐 순서〕
- [ ] 20 **Promise** — `then`/`catch`/`finally`, 상태(pending/fulfilled/rejected), 체이닝
- [ ] 21 **async / await** (ES2017) + **에러 처리 심화** — Promise를 동기처럼, `try/catch`/`throw`, **커스텀 에러 클래스**·`Error cause`(ES2022). axios 실패 처리에 직결
- [ ] 22 **여러 요청 다루기** — `Promise.all`/`allSettled`(ES2020)/`any`(ES2021), fetch & axios
- [ ] 23 **React에서 비동기** — `useEffect` + fetch + setState, 로딩/에러 상태 ([[props vs state]])

### 5단계 — 모듈 · 불변성 · 마무리
- [ ] 24 **모듈 시스템** — `import`/`export` default vs named 심화(이미 [[개념파악]]에 메모), 동적 `import()`
- [ ] 25 **불변성 패턴** — 왜 안 바꾸고 새로 만드나, `structuredClone`(깊은 복사), state 업데이트 정석
- [ ] 26 **정규식 기초** (선택) — 입력 검증(이메일/비번)에 딱 필요한 만큼만
- [ ] 27 **(선택) TS로 잇기** — 타입 기초 + 제네릭 맛보기(`reduce<...>` 이미 써봄)

> ⛏️ = **원리 심화 코너 있는 장.** 실무 수준으로 먼저 끝내고, 〔더 파기〕는 한 바퀴 돈 뒤 또는 막힐 때.
> ⚪ 가볍게 훑고 지나갈 것(읽다 만나면 당황 안 할 정도): `BigInt`·`globalThis`(ES2020), getter/setter, 태그드 템플릿 → 해당 장에 한 컷씩.
