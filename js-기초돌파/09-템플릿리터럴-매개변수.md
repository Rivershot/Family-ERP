# 09. 템플릿 리터럴 · 매개변수 — `` `${}` ``, 기본값 · rest · 구조분해 매개변수

> **목표:** 문자열 조립(`${}`)과 함수 입구의 신문법(기본값·rest·구조분해)을 자유롭게 쓴다.

## 왜 필요한가
네 코드에 이미 깔려 있다: `fmtTime`의 `` `${t.slice(0,2)}:${t.slice(2,4)}` ``, `dateLabel`의 `` `${month}월 ${day}일` ``.
그리고 React props `function Card({ title, icon })` = **구조분해 매개변수**. 함수 입구 문법은 React를 읽는 기본기.

---

## 1) 템플릿 리터럴 — 백틱 `` ` `` + `${}`
```js
const name = "채원", age = 34;
const s = `${name}님은 ${age}살`;     // "채원님은 34살"
const sum = `합계: ${1000 + 500}원`;  // ${} 안엔 "표현식" 아무거나
```
- 따옴표 + `+` 로 잇던 걸 **`${}`** 로. 가독성↑, 따옴표 지옥 탈출.
- **여러 줄** 그대로:
```js
const msg = `안녕하세요
가계부입니다`;   // 줄바꿈이 그대로 들어감
```
- `${}` 안엔 **표현식**만(문장 ❌). 삼항·함수호출·`a ?? b` 다 됨: `` `${cnt > 0 ? cnt : "없음"}` ``.

**전통 웹 비유:** `"이름: " + name + " (" + age + ")"` → `` `이름: ${name} (${age})` ``.

## 2) (⚪ 가볍게) 태그드 템플릿
함수로 템플릿을 가공하는 고급 문법. 라이브러리(styled-components의 `` styled.div`...` ``, GraphQL `` gql`...` ``)에서 본다 — **읽을 때 "아 이런 거구나" 정도**면 충분.
```js
function tag(strings, ...values) { /* strings: 글자조각, values: ${}값들 */ }
tag`이름 ${name} 나이 ${age}`;
```

## 3) 기본값 매개변수 — 안 넘기면 이 값
```js
function greet(name = "손님") { return `${name}님`; }
greet();        // "손님님"
greet("채원");  // "채원님"
```
- 인자가 **`undefined`일 때만** 기본값 발동(`null`은 그대로 null). → [[04-대입과-단축평가]]의 `??`와 같은 결.
- React: `function Card({ kind = "expense" })` 처럼 props 기본값에 자주.

## 4) rest 매개변수 — 나머지를 배열로 모음 (`...`)
```js
function sum(...nums) {          // 인자 몇 개든 nums 배열로
  return nums.reduce((a, n) => a + n, 0);
}
sum(1, 2, 3);      // 6
sum(1, 2, 3, 4);   // 10

function log(first, ...rest) {  // 앞 하나 + 나머지
  console.log(first, rest);     // 1 [2,3]
}
log(1, 2, 3);
```
- [[05-구조분해-스프레드-rest]]의 rest와 같은 원리(받는 쪽에서 **모으기**).
- ⚠️ 스프레드(펼치기)와 기호가 같다: **함수 매개변수 자리 = rest(모으기)**, **호출 인자/배열 자리 = 스프레드(펼치기)**.
```js
const arr = [1, 2, 3];
sum(...arr);   // 스프레드: 펼쳐서 sum(1,2,3)으로 호출
```

## 5) 구조분해 매개변수 — 객체를 입구에서 풀어 받기 ⭐ (React props!)
```js
// 객체 하나 받아 안에서 꺼내 쓰기 (이름순서 무관)
function dateLabel({ diff, month, day }) {
  return { 0: "Today", 1: "내일" }[diff] ?? `${month}월 ${day}일`;
}
dateLabel({ diff: 1, month: 6, day: 24 }); // "내일"

// 기본값까지 함께
function Card({ title, icon, kind = "expense" }) { /* ... */ }
```
- 호출부가 **이름으로** 넘기니(`{ month: 6 }`) 인자 순서를 안 외워도 됨 + 무엇을 넘기는지 드러남.
- **이게 React 함수 컴포넌트의 props 그 자체:** 부모가 `<Card title="예산" kind="budget" />` → React가 `{ title:"예산", kind:"budget" }` 객체로 묶어 넘김 → 입구에서 구조분해.

## ⚡ 직접 해보기
```js
const member = { name: "민지", age: 31 };
```
1. 템플릿 리터럴로 `"민지(31)"` 문자열 만들기 (`member` 사용)
2. `function pow(base, exp = 2)` — `exp` 안 넘기면 제곱. `pow(3)`, `pow(2,3)` 결과는?
3. `function max(...nums)` — 받은 숫자 중 최댓값 반환 (힌트: `Math.max(...nums)`)
4. `function intro({ name, age })` — `intro(member)` 가 `"민지는 31살"` 반환하게
5. (생각) 4번을 `function intro(member)` + `member.name`으로 쓴 것과 비교해 구조분해 매개변수의 장점은?

## ✅ 체크포인트
- `` `${표현식}` `` = 문자열 조립 + 여러 줄. `${}` 안엔 **표현식만**.
- 기본값 매개변수 = 인자가 **undefined일 때** 대체값.
- rest `(...args)` = 나머지를 **배열로 모음**(매개변수 자리). 호출부 `...arr`는 스프레드(펼치기).
- 구조분해 매개변수 `({ a, b })` = **React props**의 정체.

<summary>정답</summary>

```js
// 1
`${member.name}(${member.age})`;
// 2
function pow(base, exp = 2) { return base ** exp; }
pow(3);    // 9  (exp 기본 2)
pow(2, 3); // 8
// 3
function max(...nums) { return Math.max(...nums); }  // rest로 모아 → 스프레드로 펼쳐 전달
// 4
function intro({ name, age }) { return `${name}는 ${age}살`; }
intro(member); // "민지는 31살"
// 5: 입구에서 필요한 키(name, age)가 드러나고, member.name처럼 매번 member.를 안 붙여도 됨.
//    React props가 정확히 이 형태라 컴포넌트 입구만 봐도 무엇을 받는지 보임.
```
