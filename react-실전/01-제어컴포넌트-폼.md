# 01. 제어 컴포넌트 & 폼 — 입력값을 state로 다루기

> **목표:** 입력칸의 값을 **state가 쥐고** 있게 만들고(`value`+`onChange`), 폼 제출을 React식으로 처리한다.

## 왜 필요한가
가계부의 시작은 **입력**이다. "지출 8,000원 / 점심 / 식비" 를 받아야 목록에 쌓는다.
Dashboard는 조회 전용이라 입력이 0개였다 — 그래서 폼은 **완전히 새 개념**이다.

---

## 전통 웹과 뭐가 다른가 (WebSquare 비유)

**전통 웹:** 값은 **DOM(화면 요소)이 들고** 있다. 제출할 때 꺼내 읽는다.
```js
// 제출 순간 DOM에서 값을 긁어옴
const amount = document.getElementById("amount").value;
```

**React:** 값은 **state가 들고** 있다. 입력칸은 그 state를 **비추기만** 한다.
```jsx
const [amount, setAmount] = useState("");
<input value={amount} onChange={(e) => setAmount(e.target.value)} />
```
- `value={amount}` → 화면은 **state를 비춘다**(state가 진실의 원천, single source of truth).
- `onChange` → 타이핑할 때마다 **state를 갱신**, 그럼 리렌더되며 새 값이 다시 비침.
- 이렇게 **state가 통제하는 입력칸**을 **제어 컴포넌트(controlled component)** 라 한다.

> 핵심 사고: 전통 웹은 "필요할 때 DOM에서 **읽어온다**". React는 "값이 항상 **state에 있다**, 화면은 그걸 따라온다." (UI = f(state))

---

## 1) 입력 하나 — value + onChange 한 쌍
```jsx
function AmountInput() {
  const [amount, setAmount] = useState("");
  return (
    <input
      value={amount}                                   // state → 화면
      onChange={(e) => setAmount(e.target.value)}      // 화면 → state
      placeholder="금액"
    />
  );
}
```
- `e` = 이벤트 객체, `e.target` = 그 input DOM, `e.target.value` = 지금 입력된 글자.
- `value`만 주고 `onChange`를 안 주면 **타이핑이 안 먹는다**(state가 안 바뀌니 화면도 그대로). React 콘솔이 경고함.

## 2) 입력 여러 개 — 핸들러 하나로 (object state)
가계부 폼은 금액·내용·카테고리·날짜… 입력이 많다. 핸들러를 4개 만들지 말고 **객체 state 하나 + `name`** 으로.
```jsx
const [form, setForm] = useState({ amount: "", content: "", category: "식비" });

const handleChange = (e) => {
  const { name, value } = e.target;          // input의 name 속성으로 어느 칸인지 구분
  setForm((prev) => ({ ...prev, [name]: value }));   // ⭐ 스프레드로 나머지 유지 + 해당 키만 교체
};

<input name="amount"  value={form.amount}  onChange={handleChange} />
<input name="content" value={form.content} onChange={handleChange} />
```
- `{ ...prev, [name]: value }` → **05장 스프레드 + 동적 키**. 나머지 필드는 그대로 두고 바뀐 칸만 갱신.
- `[name]`(대괄호) = **계산된 키**. `name`이 `"amount"`면 `amount` 키를 바꾼다. (문자열 아니라 변수!)
- **왜 `setForm(prev => ...)`?** 직전 state를 안전하게 받아 그 위에 얹기 위함(함수형 업데이트). 〔원리는 02장 ⛏️〕

## 3) 제출 — onSubmit + preventDefault
```jsx
const handleSubmit = (e) => {
  e.preventDefault();          // ⭐ 폼 기본동작(페이지 새로고침) 막기
  console.log(form);           // 여기서 목록에 추가(2단계)·서버 전송
  setForm({ amount: "", content: "", category: "식비" });  // 입력칸 리셋
};

<form onSubmit={handleSubmit}>
  <input name="amount" value={form.amount} onChange={handleChange} />
  <button type="submit">추가</button>
</form>
```
- `<form onSubmit>` + 버튼 `type="submit"` → 버튼 클릭/엔터 모두 제출로 잡힌다.
- `e.preventDefault()`를 빼면 **페이지가 새로고침**돼 state가 다 날아간다(전통 form의 기본동작). React 폼의 필수 관문.

## ⚡ 직접 해보기
> 콘솔/플레이그라운드보다 **실제 가계부 컴포넌트**에 써보는 게 이 장의 목적. 작은 `ExpenseForm` 하나 만들어서:

1. 금액 입력칸 하나를 **제어 컴포넌트**로 (`value`+`onChange`, state명 `amount`)
2. 금액·내용·카테고리 **3칸을 객체 state 하나**(`form`)로 묶고 **핸들러 하나**(`handleChange`)로 처리
3. `<form onSubmit>`으로 제출 시 `console.log(form)` 찍고 **입력칸 리셋**
4. (생각) `value={form.amount}`만 쓰고 `onChange`를 안 달면 왜 타이핑이 안 될까?
5. (생각) `handleChange`에서 `setForm({ [name]: value })`로 쓰면(스프레드 없이) 무슨 일이 날까?

---

## ✅ 체크포인트
- 제어 컴포넌트 = **`value`(state→화면) + `onChange`(화면→state)** 한 쌍. 값의 진실은 state에.
- 입력 여러 개 = **객체 state + `name` + `{ ...prev, [name]: value }`**.
- 제출 = `<form onSubmit>` + **`e.preventDefault()`** (새로고침 막기) → 처리 → 리셋.
- 전통 웹은 "DOM에서 읽기", React는 "state가 들고 화면이 따라옴".

```jsx
import { useState } from "react";

function ExpenseForm() {
  // 1: 단일 제어 컴포넌트
  // const [amount, setAmount] = useState("");

  // 2: 객체 state로 여러 입력 묶기
  const [form, setForm] = useState({ amount: "", content: "", category: "식비" });

  const handleChange = (e) => {
    const { name, value } = e.target;
    setForm((prev) => ({ ...prev, [name]: value }));
  };

  // 3: 제출
  const handleSubmit = (e) => {
    e.preventDefault();
    console.log(form);                                   // 다음 단계(2단계): 목록 state에 추가
    setForm({ amount: "", content: "", category: "식비" });
  };

  return (
    <form onSubmit={handleSubmit}>
      <input name="amount"  value={form.amount}  onChange={handleChange} placeholder="금액" />
      <input name="content" value={form.content} onChange={handleChange} placeholder="내용" />
      <select name="category" value={form.category} onChange={handleChange}>
        <option>식비</option><option>교통</option><option>여가</option>
      </select>
      <button type="submit">추가</button>
    </form>
  );
}

// 4: onChange가 없으면 state가 안 바뀜 → 리렌더돼도 value는 늘 같은 state값 → 화면 고정(타이핑 무시).
//    React가 "controlled input에 onChange가 없다"고 경고.
// 5: { [name]: value }만 쓰면 그 키 하나짜리 객체로 통째 교체 → 나머지 필드(content 등)가 사라짐.
//    그래서 { ...prev, [name]: value }로 나머지를 펼쳐 유지해야 함. (불변 업데이트 = 05장 스프레드)
```
