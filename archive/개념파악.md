# 날짜별 개념 파악

그날그날 새로 알게 된 개념을 기록한다. (주제별 정리는 루트 `study.md` 참고)

---

## 2026-05-27

### 1. Tailwind는 왜 클래스를 일일이 쓰나 (utility-first 철학)
- "클래스 정의 후 이름으로 적용"하는 방식(BEM, WebSquare 등)이 **전통적인 주류** 방식. Tailwind는 이를 의도적으로 뒤집은 **유틸리티-first**.
- 유틸리티 조합이 길어 보이는 건 정상이며 의도된 설계. 장점은 별도 CSS 파일 없이 마크업만 보고 스타일 파악 가능.
- **재사용은 CSS 클래스가 아니라 컴포넌트로** 묶는 게 React식 사고. (`@apply`는 알아두되 남용 금지 — 버튼·인풋 같은 진짜 반복 요소에만)

### 2. Flexbox 정렬
- flex 아이템은 **교차축으로 기본 stretch**됨 → `self-start`(`align-self: flex-start`)로 끊으면 내용 크기만큼만 차지.
- 주축 기본은 `justify-content: flex-start`(왼쪽 쏠림) → `justify-center`로 중앙 정렬.
- "빈 flex 공간 없이 내용 크기만큼"은 폭 강제(`w-full`/고정폭/`flex-1`)를 빼고 `w-fit`으로.

### 3. border-bottom (Tailwind 표기)
- `border-b`(1px), `border-b-2`, 임의값 `border-b-[3px]`. **`border-b-1`은 없는 클래스** (무시됨).
- 색은 별도: `border-violet-700`.
- **투명 보더 트릭**: 평소 `border-b-2 border-transparent`, 선택 시 색 지정 → 밑줄 생겨도 글자가 위아래로 안 밀림.

### 4. 그라데이션 텍스트
- 원리: 글자에 그라데이션 배경을 깔고 글자 모양으로 잘라냄 → `bg-clip-text text-transparent`.
- Tailwind v4는 `bg-linear-to-br from-... to-...` (v3는 `bg-gradient-to-br`). 135deg = `to-br`(좌상단→우하단).

### 5. 탭(Tab) 기초 패턴
- 핵심은 **"지금 어떤 탭이 활성인지"를 기억하는 state** 한 줄 (`useState`).
- 배열을 `map`으로 버튼 렌더 → `onClick`에서 `setState` → 조건부 렌더링으로 내용 전환.
- **라우트 기반 탭**: 활성 여부를 `useLocation`(현재 URL)으로 판단, 내용은 `Outlet`이 담당. 열린 탭 목록만 state로 관리.

### 6. inline style 작성법
- `style={{ ... }}` — **바깥 `{}`** = JSX 표현식 시작, **안쪽 `{}`** = JS 객체.
- 속성은 객체라 **쉼표로 구분**, 이름은 **camelCase** (`fontWeight`, `backgroundColor`).
- 길면 변수로 빼서 `style={tabStyle}`. (Tailwind 프로젝트에선 가능하면 className 우선, inline은 동적 계산값에)

### 7. 구조용 시맨틱 태그
- `aside`/`section`/`header`/`nav`/`main`/`footer` 등은 **표준 내장 태그**(커스텀 아님).
- 화면엔 `div`처럼 블록 박스로 보이고, **차이는 의미(접근성·가독성)뿐** — 기능/동작 없음.
- 반면 `a`(앵커)/`button`/`input`은 **실제 동작을 가진 태그**라 별개 분류.
- 한 줄 요약: *"구조용 시맨틱 태그 = 보기엔 div, 의미만 다른 표준 태그 / 앵커·버튼 = 진짜 동작하는 태그."*

### 8. Tailwind border-radius (`rounded-`)
- 스케일: `rounded-none`(0) / `xs`(2px) / `sm`(4) / `md`(6) / `lg`(8) / `xl`(12) / `2xl`(16) / `3xl`(24) / `4xl`(32) / `full`(완전 둥글게)
- 임의값: `rounded-[25px]` (스케일에 없는 값)
- **부분만 둥글게**: 변 단위 `rounded-t/b/l/r-*`, 모서리 단위 `rounded-tl/tr/br/bl-*` (예: `border-radius: 0 5px 5px 0` → `rounded-r-[5px]`)
- ⚠️ v4에서 이름이 한 칸 밀림: v3 `rounded`(4px) → v4 `rounded-sm`, v3 `rounded-sm`(2px) → v4 `rounded-xs`. v4엔 접미사 없는 `rounded` 안 씀.
